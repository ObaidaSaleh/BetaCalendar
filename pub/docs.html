Getting started / Installation

<script defer type="text/javascript" src='BetaCalendar.js'></script>

BetaCalendar offers various organizational-based widgets with simple to implement customization.

To use BetaCalendar you must first include two javascript files at the top of your page, one is the BetaCalendar library, and the other is jquery. (jquery version 3.5.1 was used for the creation of BetaCalendar)

Basic Implementation

                    
const calendar1 = new BetaCalendar("#normalCalendar");

Initializaing a basic version of BetaCalendar is very simple. The basic version has the base colour scheme, and no extended functionality such as the TodoList or the minimizable features.

To use BetaCalendars basic module, create a new instance of BetaCalendar using it's constructor and pass in the id of the component that you would like to be the trigger for toggling the module.

Constructor

                    
const calendar1 = new BetaCalendar("#normalCalendar");
const calendar2 = new BetaCalendar("#minimizableCalendar", false, true, "basic", true);
const calendar3 = new BetaCalendar("#miniAndDragCalendar", true, true, "basic", true);
const calendar4 = new BetaCalendar("#importedRemindersCalendar", true, true, "basic", true);

const calendar5 = new BetaCalendar("#nightCalendar", true, true, "basic", true,
{
main: "#006bb3",
secondary: "#404040",
background: "#121212",
cells: "#181818",
text: "white"
});

The BetaCalendar constructor can take various parameter inputs to customize the calendar:


Parameter Type Default Functionality
selector String Required Parameter The id of the desired component to attach calendar toggle functionality to. Must include "#" at the beginning for ids.
draggable Boolean false Specifies whether a minimized calendar is draggable or not. If enabled, the minimized calendar can be dragged around and also pinned.
minimizable Boolean false Specifies whether a calendar is minimizable or not. If enabled, allows a calendar to be minimized into a more condensed version.
type String "basic" Specifies the type of widget for the calendar module, current values are only "basic".
todoListEnabled Boolean false Specifies whether a calendar has a todoList component or not. When enabled, the calendar will have a button on the bottom right that transforms the widget into a todoList.
theme theme Object See below Specifies the colour scheme for the calendar, divided into "main", "secondary", "background", "cells", "text".

theme parameter

                        

{
    main: "#006bb3",
    secondary: "#404040",
    background: "#121212",
    cells: "#181818",
    text: "white"
}

The BetaCalendar constructor can also take a theme parameter to specify the colour scheme of the widget:


Theme Section Type Default Functionality
main String "#99d6ff" Controls the main colour for the important parts of the calendar, such as the current day of the month, and the current month and year header.
secondary String "#a6a6a6" Controls the secondary colour for the calendar, components such as days of the week under the header.
background String "#E6E6F2" Controls the background colour for the calendar module.
cells String "white" Controls the background colour for the main cells.
text String "black" Controls the colour of the text.

BetaCalendar functions that developers can use to interact with the calendar modules:


Function Arguments Return Description
importReminders() reminders[] (Reminder Object) - Allows the developer to preset the reminders for a specific calendar, must be in the form of a reminder object as seen below.
exportReminders() - Reminder Object Allows the developer to export the current reminders in a calendar to be used elsewhere, is in the form of a Reminder Object array as seen before

Reminder Object

Field Type Description
date JavaScript Date() Object The date of the reminder.
label String The label for the reminder.
urgency Number : 1 || 2 || 3 The urgency of the reminder, can be either 1 for low, 2 for medium, or 3 for high.
